1.2 Python Building Blocks


1. Identifiers

An identifier is a name given to variables, functions, classes, etc. It must follow these rules:

  • Must start with a letter (a-z, A-Z) or underscore (_).
  • Can contain letters, digits (0-9), and underscores.
  • Cannot use Python keywords.
  • Special characters (!, @, #, $, etc.) are not allowed.

2. Keywords

KThe keywords are special words reserved for some purpose. For instance a variable name can not be from because it is a keyword.

Examples: 

False, None, True, and, as, assert, break, class, continue,
def, del, elif, else, except, finally, for, from, global, if, import

3. Variables

A variable is nothing but a reserved memory location to store values. Variable is an entity to which the programmer can assign some values. Programmer choose the variable name which is meaningful.

Rules of Variables:

  • Follow identifier rules.
  • Use assignment operator (=) to assign values.
  • Case-sensitive (e.g., myVar ≠ MyVar).


Examples:

x = 10  # Integer variable
y = "Hello"  # String variable

4. Indentation

Leading white space at the beginning of the logical line is called indentation. Python programs get structured through indentation, i.e. code blocks are defined by their indentation.

Examples:

if True:
    print("Indented code block")

5. Comments

Comments are the kind of statements that are written in the program for program understanding purpose. By the comment statements it is possible to understand what exactly the program is doing. Comments are generally used to explain the code.


  • Single-line: 
    # print is a statement 
print( ‘Hello Python? )

  • Multi-line: 
    ’’’ this is multiple line comment 
This is the program of python ’’’
print(“hello world")
