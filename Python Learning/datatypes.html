1.5 Data Types in Python


The type of data value that can be stored in an identifier/variable is known as its data type. In programming, data types are an important concept. A variable can store data of different types, and different types can do different things. Data types are used to define the type of variable.


Common Data Types in Python

Python has several built-in data types that you can use to store different kinds of data. Here are five of the most commonly used ones:


    1. Numeric:-

    Python's numeric data types are used to hold numeric values. Here are some common numeric types:


    • int:

      • Represents signed integers of non-limited length. Integers are whole numbers, positive or negative, without decimals.

    • float:

      • Represents floating-point numbers, which are numbers with a decimal point. It's accurate up to 15 decimal places by default.

    • complex:

      • Represents complex numbers, which are numbers of the form a + bi, where a and b are real numbers, and i is the imaginary unit (the square root of -1).


    Example:

                    # Creates a variable with integer value
                a = 5
                print("Type of a:", type(a))   
      
                # Creates a variable with float value
                b = 5.0
                print("Type of b:", type(b)) 
      
                # Creates a variable with complex value
                c = 2 + 4j
                print("Type of c:", type(c))

2. String:-

A string is a collection of characters enclosed in single quotes ('), double quotes ("), or triple quotes (""" or '''). It can contain any number, letter, or symbol.


Example:

           
                str = "Hello I'm Tanvi!"
                print("String is:", str)  # Output: String is: Hello I'm Tanvi !

3. List:-

A list is an ordered, mutable collection of elements similar to arrays in other languages. It can hold different data types within the same list. Elements are enclosed in square brackets ([]) and separated by commas (,).


Example:


                # List of only integers
                a = [10, 30, 20, 50]
                print(a)  # Output: [10, 30, 20, 50]
      
                # List of only strings
                b = ['tue', 'fri', 'mon']
                print(b)  # Output: ['tue', 'fri', 'mon']

4. Tuple:-

A tuple is an ordered, immutable collection of elements. It is similar to a list, but it is enclosed in parentheses () instead of square brackets. Tuples are generally faster than lists.


Example:

                
                # Tuple of only integers
                a = (10, 30, 20, 50)
                print(a)  # Output: (10, 30, 20, 50)
      
                # Tuple of only strings
                b = ('tue', 'fri', 'mon')
                print(b)  # Output: ('tue', 'fri', 'mon')
      
                # Tuple of both integers and strings
                c = ('tue', 1, 'fri', 4, 7, 'mon')
                print(c)  # Output: ('tue', 1, 'fri', 4, 7, 'mon')

5. Dictionary:-

A dictionary is an unordered collection of data in the form of key-value pairs. It is enclosed in curly braces ({}) and each pair is separated by a colon (:). Dictionaries are mutable, meaning you can change their values after they are created.


Example:

                a = {1: 'Red', 2: 'Blue', 3: 'Green'}
      
                # Prints the keys and values
                print(a)  # Output: {1: 'Red', 2: 'Blue', 3: 'Green'}
      
                # Prints the keys of the dictionary
                print(a.keys())  # Output: dict_keys([1, 2, 3])
      
                # Prints the values of the dictionary
                print(a.values())  # Output: dict_values(['Red', 'Blue', 'Green'])

Input through Keyboard

In Python, you can input data from the user using the `input()` function.


Syntax:

            variable_name = input([prompt])

where `prompt` is an optional string that will be displayed to the user before they enter their input.


Example:

            a = input("Enter a number: ")
            print(a)  # Output: The number entered by the user
